# Чек-лист для самопроверки

## Тестирование

### Ручное тестирование
Запустите программу с разными наборами аргументов, проверьте корректность работы и обработку ошибок. Примеры приведены для каждого варианта.

### Проверка утечек памяти
Используйте `valgrind`:
```
valgrind --leak-check=full ./matrix sum 2 2 1 2 3 4 5 6 7 8
```
Убедитесь, что все выделенные блоки освобождены.

### Сравнение с эталоном
Для проверки правильности работы можно сравнить вывод вашей программы с выводом программы, использующей стандартные функции (например, `printf`, `atoi` и т.п.), но в рамках лабораторной работы это не обязательно.

## Чек-лист

- [ ] Программа компилируется без ошибок и предупреждений с флагами `-Wall -Wextra -Werror`.
- [ ] В `Makefile` реализованы цели `all`, `clean`, `fclean`, `re`.
- [ ] Программа корректно разбирает `argc` и `argv`, преобразует строки в числа через `ft_atoi`.
- [ ] Реализованы базовые функции: `create_matrix`, `remove_matrix`, `print_matrix` с использованием структуры `matrix_t`.
- [ ] Операции по варианту (сложение/вычитание, умножение на число/транспонирование, умножение матриц или сравнение/анализ свойств) работают корректно.
- [ ] Все операции с выводом выполняются через функции libft (`ft_putchar_fd`, `ft_putstr_fd`, `ft_putnbr_fd`, `ft_putendl_fd`).
- [ ] Программа корректно обрабатывает ошибки (неверные размеры, несовместимые матрицы, невозможность выделить память) и выводит сообщение в `stderr`.
- [ ] Нет утечек памяти (`valgrind` показывает "All heap blocks were freed").
- [ ] Код отформатирован, нет глобальных переменных, нет лишних включений заголовочных файлов.

## Критерии оценивания (максимум 5 баллов)

| Критерий | Баллы |
|----------|-------|
| Программа корректно разбирает аргументы командной строки (`argc`/`argv`), преобразует числа через `ft_atoi` и обрабатывает ошибки (недостаток аргументов, некорректные размеры, несовместимые матрицы) | 1 |
| Реализованы базовые функции: `create_matrix`, `remove_matrix`, `print_matrix` с использованием структуры `matrix_t` (корректное выделение и освобождение памяти) | 1 |
| Операции по выбранному варианту реализованы и работают корректно (сложение/вычитание, умножение на число/транспонирование, умножение матриц или сравнение/анализ свойств) | 1 |
| Вывод результатов и сообщений осуществляется только через функции `libft` (`ft_putchar_fd`, `ft_putstr_fd`, `ft_putnbr_fd`, `ft_putendl_fd`), запрещённые стандартные функции (`printf`, `puts`, `atoi`, `malloc` и т.д.) в `main.c` не используются | 1 |
| Отсутствие утечек памяти (проверка `valgrind`), корректное освобождение всей выделенной памяти | 1 |
| **Итого** | **5** |

### Штрафы:
- Использование любой запрещённой стандартной функции (например, `printf`, `strlen`, `atoi`, `malloc`, `free` и т.п.) в `main.c` — работа не принимается (**0 баллов**).
- Утечки памяти — **−1 балл**.
- Отсутствие обработки ошибок (неверные аргументы, несовместимые размеры, невозможность выделить память) — **−1 балл**.
- Программа не компилируется с флагами `-Wall -Wextra -Werror` — **−5 баллов**.
